# Sub-Agent 使用规范

---

## 概述

Sub-Agent 是可选的执行能力。系统采用**能力检测 + 自动降级**机制：

```
任务到达
    ↓
检测 Sub-Agent 可用性
    ↓
├─ 可用 ──→ 委托 Agent 执行（并行、独立决策）
│
└─ 不可用 ──→ 调用 Skill 执行（标准化流程）
    ↓
产出格式保持一致
```

---

## 何时使用

以下场景触发 Sub-Agent 检测：

- 用户明确说"委托: xxx"
- 任务标记为支持 Agent 执行
- 需要并行处理多个任务

---

## 核心原则

### 1. 文件通信

- Sub-Agent 之间通过**文件**传递数据
- 不要把大段内容放进返回消息
- 分析报告写入 `workspace/`

### 2. 保护上下文

主 Agent 上下文只接收：

- 状态（成功/失败）
- 文件路径
- 需要决策的问题（简短）

不要接收：

- 完整代码
- 大段文档
- 原始数据

### 3. 返回规范（4 行）

```
状态：已完成 / 失败 / 需要决策
报告：workspace/X.Y-xxx.md
产出：N 个文件（列出路径）
决策点：[如有，一句话描述]
```

---

## 任务编号

| 编号 | 含义 |
|------|------|
| X.0 | 主 Agent 产出（规划、决策） |
| X.1, X.2... | 第 X 阶段的 Sub-Agent 任务 |
| 同一 X | 可并行执行 |
| X 递增 | 串行依赖 |

---

## 委托模板

```
任务 {X.Y}: {任务名}

目标：{清晰的单一职责}

输入：
- {文件路径或上下文}

产出：
- 代码：{写入路径}
- 报告：workspace/{X.Y}-报告.md

要求：
- 使用 Write 工具写入文件
- 只返回 4 行摘要
```

---

## 委托触发条件

### 自动委托（推荐）

| 场景 | 执行方式 | 理由 |
|------|----------|------|
| 单一规范任务 | Skill 直接执行 | 流程标准化，无需决策 |
| 复杂并行任务 | Agent 委托 | 需要独立决策和并行处理 |
| 调研查询任务 | Agent 委托 | 需要独立查询和分析 |

### 用户显式委托

```
用户: "委托：生成代码"
→ 创建 Agent 执行

用户: "生成代码"
→ 主 Agent 判断（可能使用 Skill 或委托 Agent）
```
